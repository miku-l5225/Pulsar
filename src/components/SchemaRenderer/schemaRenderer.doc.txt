
### `WrappedSelect`

#### **用途**

该组件将 `shadcn-vue` 的复合式 `Select` 组件封装成一个单一的、数据驱动的组件。它简化了 `Select` 的使用，使其可以通过传入一个结构化的 `groups` prop 来动态生成所有选项和分组，并支持 `v-model` 双向绑定。这使得它非常适合用于动态表单或如 `schemaRenderer` 这样的配置化 UI 渲染器中。

#### **Props**

*   **`modelValue`**:
    *   **类型**: `String | Number`
    *   **是否必需**: 否
    *   **默认值**: `undefined`
    *   **描述**: 当前选中的值，用于 `v-model`。

*   **`placeholder`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **默认值**: `undefined`
    *   **描述**: 当没有选中任何值时，在选择框中显示的占位提示文本。

*   **`groups`**:
    *   **类型**: `Array<SelectGroupOption>`
    *   **是否必需**: 是
    *   **描述**: 一个用于定义下拉选项的数组。数组中的每个对象代表一个分组 (`<SelectGroup>`)，包含一个可选的 `label` 字符串（分组标题）和一个必需的 `items` 数组（该分组下的选项列表）。每个 `item` 对象都必须有 `value`（选项的实际值）和 `label`（选项的显示文本）。

### `EnhancedSlider`

#### **用途**

这是一个增强型的滑块（Slider）组件。它在标准滑块的右侧增加了一个数字输入框，允许用户通过拖动滑块进行快速调节，也可以通过输入框进行精确的数值设定。

#### **Props**

*   **`modelValue`**:
    *   **类型**: `Number`
    *   **是否必需**: 否
    *   **默认值**: `0`
    *   **描述**: 滑块的当前数值，用于 `v-model`。

*   **`min`**:
    *   **类型**: `Number`
    *   **是否必需**: 否
    *   **默认值**: `0`
    *   **描述**: 滑块允许的最小值。

*   **`max`**:
    *   **类型**: `Number`
    *   **是否必需**: 否
    *   **默认值**: `100`
    *   **描述**: 滑块允许的最大值。

*   **`step`**:
    *   **类型**: `Number`
    *   **是否必需**: 否
    *   **默认值**: `1`
    *   **描述**: 滑块拖动或输入框调整时的步长。

*   **`showInput`**:
    *   **类型**: `Boolean`
    *   **是否必需**: 否
    *   **默认值**: `true`
    *   **描述**: 控制右侧的数字输入框是否显示。

*   **`allowExceedMax`**:
    *   **类型**: `Boolean`
    *   **是否必需**: 否
    *   **默认值**: `false`
    *   **描述**: 如果设置为 `true`，则允许在输入框中输入超过 `max` 属性设定的值。滑块本身仍会受限于 `max`。

*   **`class`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **描述**: 应用于组件根元素的额外 CSS 类。

### `PopableTextarea`

#### **用途**

一个可弹出的多行文本输入框。在常规状态下，它是一个标准的 `Textarea`；当用户鼠标悬停时，右上角会显示一个“展开”按钮，点击后会弹出一个对话框（Dialog），提供更大的编辑空间，非常适合编辑长文本或代码片段。

#### **Props**

*   **`modelValue`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **默认值**: `undefined`
    *   **描述**: 文本框的内容，用于 `v-model`。

*   **`placeholder`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **默认值**: `undefined`
    *   **描述**: 文本框的占位提示文本。

*   **`dialogTitle`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **默认值**: `"编辑内容"`
    *   **描述**: 弹出对话框顶部显示的标题文本。

*   **`class`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **描述**: 应用于组件根元素（`div` 包装器）的额外 CSS 类。

### `RegexEditor`

#### **用途**

该组件提供了一个功能丰富的界面，用于管理和编辑一组正则表达式替换规则 (`RegexRule`)。它整合了拖拽排序、独立的折叠面板以及行内快捷操作，让复杂的规则配置变得直观和高效。每个规则项都可以独立展开进行详细编辑，或在折叠状态下进行快速启用/禁用和删除操作。

#### **核心功能**

*   **可拖拽排序**: 基于 `vuedraggable`，用户可以通过拖动每个规则项左侧的拖拽手柄来轻松调整规则的应用顺序。
*   **折叠式布局**: 每个规则都包裹在 `shadcn-vue` 的 `Accordion` 组件中，默认收起以保持界面整洁。标题行清晰地展示了规则名称、启用状态开关和操作按钮。
*   **行内快捷操作**: 在折叠状态下，用户可以直接通过 `Switch` 组件启用或禁用规则，或点击垃圾桶图标快速删除规则。
*   **详细编辑视图**: 展开 `Accordion` 后，会显示完整的编辑表单，包括用于“查找”和“替换”的输入框、用于指定应用时机（渲染时/发送时）的复选框，以及用于控制匹配深度的数字输入框。

#### **Props**

*   **`modelValue`**:
    *   **类型**: `Array<RegexRule>`
    *   **是否必需**: 是
    *   **描述**: 组件要编辑的正则表达式规则数组，用于 `v-model`。数组中的每个对象都应符合 `RegexRule` 的结构定义。

#### **Events**

*   **`update:modelValue`**:
    *   **载荷**: `Array<RegexRule>`
    *   **描述**: 当规则列表发生任何变化时（例如，重新排序、删除规则、或修改了规则的任何属性），该事件会被触发，并回传更新后的完整规则数组。这是实现 `v-model` 双向绑定的关键。

### `JSCodeEditor`

#### **用途**

该组件是一个封装了 Monaco Editor（VS Code 的核心编辑器）的、功能强大的代码编辑器。它被设计为一个完全受控的组件，通过 `v-model` 与父组件进行数据双向绑定。其核心特性是为编辑器实例注入了一套预定义的 TypeScript 类型声明，从而为在一个特定执行上下文中编写代码提供了丰富的智能感知（Intellisense）和自动补全功能。它使用 `shadcn-vue` 的 `Card` 组件进行 UI 封装，非常适合用于需要用户编写在特定环境下运行的脚本的场景，例如低代码平台的自定义逻辑编辑器或动态脚本生成器。

#### **核心功能**

*   **受控组件**: 完全通过 `v-model` 进行控制，父组件拥有对编辑器内容的唯一数据源，确保了状态管理的清晰和可预测性。
*   **强大的代码编辑体验**: 基于 Monaco Editor，提供了语法高亮、代码折叠、自动格式化等现代代码编辑器所具备的一流功能。
*   **上下文智能提示**: 组件会自动注入一个虚拟的全局类型声明文件 (`global.d.ts`)。这使得编辑器能够识别特定的全局变量（如 `ai`, `apiWrapper`）、函数和类型，并在用户编写代码时提供实时的自动补全和类型检查，极大地提升了开发效率和代码质量。
*   **简洁的 UI 封装**: 使用 `shadcn-vue` 的 `Card` 组件作为容器，提供了一个包含标题和描述的、外观统一且简洁的界面。
*   **可配置化**: 组件的标题和描述可以通过 props 进行定制，使其能够灵活地复用于应用的不同部分。

#### **Props**

*   **`modelValue`**:
    *   **类型**: `String`
    *   **是否必需**: 是
    *   **描述**: 编辑器中的代码内容，用于 `v-model` 双向绑定。

*   **`title`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **默认值**: `"JavaScript 代码编辑器"`
    *   **描述**: 显示在 `Card` 组件头部的标题文本。

*   **`description`**:
    *   **类型**: `String`
    *   **是否必需**: 否
    *   **默认值**: `"在这里编写和编辑您的代码。"`
    *   **描述**: 显示在标题下方的描述性文本。

#### **Events**

*   **`update:modelValue`**:
    *   **载荷**: `String`
    *   **描述**: 当编辑器内的代码内容发生任何变更时触发。该事件会回传更新后的完整代码字符串，这是实现 `v-model` 双向绑定的关键。
